# Contributing to QAP

Thank you for your interest in the Quiesce-Authorize Protocol.

This repository is a **research specification and intellectual property record**.
Contributions are welcome in the form of feedback, peer review, and academic
discussion — not code submissions at this stage.

## How You Can Contribute

### Research Feedback
Open a GitHub Issue using the **Research Feedback** template to:
- Point out gaps in the threat model
- Suggest related work or prior art references
- Propose protocol enhancements or alternative approaches
- Question design decisions or assumptions

### Corrections
If you find a factual error, broken reference, or typographical issue in the
specification or documentation, open an Issue with the **Bug Report** template.

### Academic Discussion
You are welcome to cite this work in your own research. See [CITATION.cff](./CITATION.cff)
for the preferred citation format.

## What Is Not Accepted

- Pull requests that modify the core protocol specification without prior
  discussion with the author
- Derivative works or re-publications of any part of this repository without
  explicit written permission from the author (see [LICENSE](./LICENSE))
- Commercial use of any kind without a separate commercial license agreement
