# Security Policy

## Scope

This repository contains the QAP (Quiesce-Authorize Protocol) specification,
whitepaper, and associated research documentation. It does not contain
production code or deployable software at this time.

## Reporting a Vulnerability or Design Flaw

If you identify a security vulnerability, logical flaw, or exploitable weakness
in the QAP protocol specification or its described architecture, please report
it responsibly.

**Do not open a public GitHub issue for security-related findings.**

### Contact

Report findings directly to the author via email:

**CHALLAYOGESWAR**
📧 challayogeswar8790@gmail.com

Please include in your report:

- A clear description of the flaw or vulnerability
- The specific section(s) of the specification affected
- A proof-of-concept or attack scenario (if applicable)
- Your suggested mitigation or fix (if any)

### Response Timeline

| Action | Timeline |
|---|---|
| Acknowledgement of report | Within 7 days |
| Initial assessment | Within 14 days |
| Resolution or mitigation published | Within 60 days |

### Recognition

Researchers who responsibly disclose valid protocol flaws will be acknowledged
in the specification changelog and any subsequent published revisions, with
their consent.

## Protocol Design Philosophy

QAP is a research specification. All design decisions, threat mitigations, and
identified flaws are documented transparently in the repository. Peer review
and critical analysis are actively welcomed through the responsible disclosure
process above.
